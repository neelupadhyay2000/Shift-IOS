Live-Shift Testing — Step by Step

Plain-English scripts for testing what happens to a running event when you add minutes to a block. No code. Just tap, watch the clock, compare.

What you're testing

During a live event you can only add time to the block that's running right now. The “Shift Timeline” button gives you +5 +10 +15. When you add minutes, the current block gets longer and everything after it slides later — unless a locked block is in the way.

The 6 rules every test checks
  1. Locked blocks never move. A locked (pinned) block is a hard wall. Adding time to another block can never push it, shrink it, or change its time.
  2. Everything after slides by the same amount. Add 10 minutes and every later block starts 10 minutes later — until it reaches a locked block.
  3. Things squeeze to fit before a lock. If the slide would push blocks past a locked block, they shrink instead, and the last one ends exactly at the locked block's start time. The lock doesn't budge.
  4. No room means no change. If there isn't enough space before the next locked block, the app refuses the whole thing — nothing moves — and tells you the most you can add.
  5. The running block keeps its start. When you add time, only the end of the current block moves later. Its start time stays put.
  6. Finished blocks stay frozen. Any block you've already marked done never changes.

How to get live in the app: pick a template → set the event's start time → open it → tap Go Live. The first block becomes the running block. Use slide to advance to mark the running block done and move to the next one. Tap Shift Timeline (clock icon, top-right) to add minutes to whatever's running.

Quick pass/fail checklist

Tick these off after a round of testing. Saved on this device.

  • ✓A locked block never changed its time, no matter what I added elsewhere.
  • Every block after the one I extended slid later by the same number of minutes (until it hit a lock).
  • When blocks squeezed before a lock, the last one ended exactly at the lock's start time.
  • When there was no room, the app refused and nothing moved.
  • The block I extended kept its start time — only its end moved.
  • Blocks I'd already marked done never changed.
