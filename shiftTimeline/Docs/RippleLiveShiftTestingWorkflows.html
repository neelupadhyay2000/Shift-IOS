SHIFT · Tester Checklist

Live-Shift Testing — Follow & Verify

Five short scripts. Each one tells you exactly what to do, what to press, and the clock time every block should read afterwards. Do it on your phone, then check it against the tables below.

Before you start

During a live event you can only add time to the block that's running right now. Tap Shift Timeline (clock icon, top-right) and you get +5 +10 +15. To pick which block is running, use slide to advance — it marks the running block done and moves to the next one. Set up each test by creating the named template, setting its start time, opening it, and tapping Go Live.

  1. A locked block never moves. Adding time elsewhere can't push it, shrink it, or change its time.
  2. Everything after slides by the same amount. Add 10 minutes → every later block starts 10 minutes later, until it reaches a locked block.
  3. Blocks squeeze to fit before a lock. If they'd run past a locked block, they shrink instead — and the last one ends exactly when the locked block starts.
  4. No room = no change. If there isn't space before the next lock, the app refuses the whole thing and nothing moves.
  5. The running block keeps its start. Only its end moves later.
  6. Finished blocks stay frozen. Anything already marked done never changes.
